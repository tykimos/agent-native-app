# Agent Native App
## 에이전트 네이티브 앱 (ANA)

**나만의 운영(運營)을 위한 하네스를 직접 만든다**

대시보드를 **보면서**, 코딩 에이전트와 **대화하며** 운영하는 앱

> "이미 제 삶 깊숙이 들어온 컨셉입니다."

---

## 문제: 앱은 넘치는데, '내 운영'은 여전히 손이 간다

- 일정·지표·주문·작업 큐… 매일 보는 건 정해져 있는데,
- **딱 내 방식대로** 보고 바꾸려면 항상 부족하다.
- 새 기능이 필요하면? → 개발 사이클을 기다린다.
- 자동화하려 챗봇을 붙이면? → **글로만** 주고받아 답답하다.

**1인 운영자(솔로프리너)에게 필요한 건 '내 손에 맞는 운영 도구' — 그것도 지금 당장.**

---

## 두 갈래의 한계

| | 전통 앱 / SaaS | 순수 챗봇 · 원격조종 |
|---|---|---|
| 조작 | 정해진 기능만 | 텍스트 명령 |
| 맥락 | 화면은 있지만 고정 | **화면 없이 눈 감고** 명령 |
| 변경 | 개발팀·릴리스 필요 | 데이터만, 앱 자체는 못 바꿈 |
| 내 방식 | 앱에 내가 맞춘다 | 매번 설명해야 |

→ **둘 다 "내가 앱에 맞추는" 구조.**

---

## ANA 정의

> **Agent Native App** = 코딩 에이전트를 **운영의 코어(런타임)** 로 삼아,
> 사용자가 **대시보드를 보면서 대화로 운영**하고,
> 필요하면 **앱 자체를 그 자리에서 바꿔** 나가는 앱.

- 에이전트는 '부가 기능'이 아니라 **앱을 움직이고 진화시키는 엔진**.
- 운영자는 **보고(watch) + 말한다(converse)**. 나머지는 에이전트가.
- **앱이 나에게 맞춰 자란다.**

---

## 3원칙

### ① 보면서 대화한다 (Watch + Converse)
시각적 상태(대시보드) + 자연어 대화를 **한 화면**에서.

### ② 에이전트가 런타임이다 (Agent as Runtime)
에이전트가 데이터를 **읽고 → 행동하고 → 앱 코드까지** 고친다.

### ③ 내 하네스를 소유한다 (Own Your Harness)
의존성 0·셀프호스팅·내 삶에 맞춰 **계속 진화**하는 나만의 운영 도구.

---

## 구성도

```
   사용자(휴대폰/PC)
        │  ① 대시보드를 보며  ② 대화로 요청
        ▼
 ┌──────────────────────┐      ┌───────────────────────────┐
 │  대시보드 (웹)        │◀────▶│  채널 (fakechat 등)        │
 │  · 핵심 상태 한눈에   │ 푸시 │  · 메시지를 에이전트로 전달 │
 │  · 채팅 입력/응답     │      └─────────────┬─────────────┘
 └──────────┬───────────┘                    ▼
            │  데이터 API           ┌───────────────────────────┐
            └──────────────────────▶│  코딩 에이전트(Claude Code)│
              읽기/제안/적용         │  · 상태 읽기 · 제안·적용     │
                                    │  · 앱 코드까지 수정         │
                                    └───────────────────────────┘
```
**인바운드는 채널로 자동 푸시, 응답은 대시보드로(제안→승인) → 보면서 대화.**

## 무엇이 다른가

| 축 | 전통 SaaS | 챗봇 | 노코드 | **ANA** |
|---|---|---|---|---|
| 시각 맥락 | O(고정) | X | O(템플릿) | **O(내 대시보드)** |
| 자연어 운영 | X | O | △ | **O** |
| 앱을 그 자리서 변경 | X | X | △ | **O(에이전트가 코딩)** |
| 내 방식에 수렴 | X | △ | △ | **O** |
| 소유·이식 | X | X | X | **O(셀프호스팅)** |

---

## 사례 — "우리집 주간 계획표"

- **대시보드**: 토스 스타일 격자(시간 × 요일 × 가족 4명), 한눈에 보는 진행률.
- **대화**: 하단 `[대화하기]` → "오늘 일정은?" / "민준 7시 영어 추가".
- **흐름**: 채팅 → 채널 → **Claude 세션**이 `schedule.json` 읽고 → **전/후 미리보기 + 승인 카드** → 누르면 반영, 모든 기기 동기화.

> 휴대폰만 있으면 **어디서든** 가족 운영. 무인증·셀프호스팅·의존성 0.

---

## 핵심 — "말하면 앱이 바뀐다"

이 앱은 쓰는 도중에 **요청만으로** 이렇게 자랐다:

- "음성으로 입력되게" → 🎙️ 마이크(Web Speech) 추가
- "30분 단위로, 한 줄·고정 높이" → 격자 규칙 변경
- "이전 5일·앞으로 5일" → 롤링 윈도우
- "하단에 [스케줄등록][대화하기] 버튼" → 액션 바 신설
- "9시 반까지 늘려줘"(가족이 채팅으로) → 표시 범위 확장

**개발 사이클이 아니라, 대화 한 번.** 이게 ANA의 진짜 힘.

---

## 왜 솔로프리너에게 중요한가

- **운영 비용 0에 수렴** — 1인이 휴대폰으로 전부 운영.
- **앱이 나와 함께 성장** — 필요할 때 말하면 그 자리서 배포.
- **하나의 하네스, 여러 도메인** — 일정·지표·주문·작업 큐, 심지어 PC방 시간 제어(timeguard)까지 같은 패턴으로.
- **완전한 소유** — 셀프호스팅·이식 가능·종속 없음.

## 어떻게 만드나 — 2개의 재사용 스킬

**`ai-solopreneur-skills`** (github.com/tykimos/ai-solopreneur-skills)

1. **`uxui-design-system`** — 토스 스타일·의존성 0 디자인 시스템(토큰·하단 액션 바·바텀시트/팝업).
2. **`fakechat-dashboard-agent`** — 대시보드 + 채널 + 코딩 에이전트로 ANA를 짓는 템플릿(서버·릴레이·아키텍처).

> "보면서 대화하는 운영 도구"를 누구나 자기 도메인에 복제할 수 있도록.

## 시작하기

1. **핵심 화면 1개** 정의 — 매일 봐야 할 상태.
2. **대시보드 + 채널 + 에이전트** 연결(템플릿 복붙).
3. 쓰면서 **말로 키운다** — 기능은 대화로 추가.
4. 외부 공개는 터널(+고정 주소)로.

**오늘의 운영을, 오늘의 대화로.**
